There are many built in functions in R such as log or exp. Using these functions in R is not much different than using them on your calculator. The function log has two arguments, one is required and one is set to a default, \(log_{b}(x)\) where b is the base. The default is b which is set to exp(1) (i.e. the natural logarithm).

Below are some examples of built in functions in R. Run them from the code box below.

exp(2)
log(8)
log(8,base=2)

Common errors

Math errors and typos are common. The error messages in R aren’t always clear. Run the code below to see the error message, then fix the code to remove the error.

2 + 5*(8^3)- 3*log10)

Uh oh, we got an Error. Nothing to worry about, errors happen all the time. Put a open parenthesis ( before the 10 in log10 to fix it and try again.

Here is another example of a common error. 
2 + 5(3)

R needs to be told to multiple 5(3) try putting an * in between the 5 and (3).

Storing results

Most of the time we will want to save our results. To do so we use the assignment operator <- notation. This lets us save a value into an object, which we can then use later similar to a variable \(x\) in algebra. The code has ran successfully if you do not receive any messages.

 height <- 62
 height <- 62

Assigned objects aren’t printed automatically. You can wrap the command in parenthesis to have R print out the results. Run the code below to see the objects printed.

(height <- 62)
(height <- 62)

Vectors

A vector is a list of values in order to be able to work with them. For us they will usually represent data collected on a characteristic of the population. In general, we want to give the vector a name so that we can call it later when needed.

We use the c() function to combine numbers into a vector. 
(a <- c(1,3,6,8,9))
(a <- c(1,3,6,8,9))
Sequence of odd numbers from 1 to 10, 
(odds <- seq(1, 10, by=2))
(odds <- seq(1, 10, by=2))

We can use the rep function to repeat a sequence of numbers in varying patterns. Type?rep in the code box and hit Run code for a text explanation of the differences.

Indexing vectors

We can index vectors to pull off values at a particular position on a vector.

Returns first and second number of the vector “primes”. 
primes <- c(2,3,5,7,11)
primes[1]
primes[2]
primes <- c(2,3,5,7,11)
primes[1]
primes[2]
Returns first three numbers of a vector 
primes <- c(2,3,5,7,11)
primes[1:3]
primes <- c(2,3,5,7,11)
primes[1:3]

We can change a vector to TRUE and FALSE by writing a logical statement.

primes <- c(2,3,5,7,11)
primes > 6
primes <- c(2,3,5,7,11)
primes > 6
This is super useful to do things like identify the numbers in primes that are greater than 6 
primes <- c(2,3,5,7,11)
primes[primes>6]
primes <- c(2,3,5,7,11)
primes[primes>6]
And count the number of primes greater than 6 
primes <- c(2,3,5,7,11)
sum(primes>6)
primes <- c(2,3,5,7,11)
sum(primes>6)

Logical Operators

Is 4 a prime? 
primes <- c(2,3,5,7,11)
primes[primes == 4]
primes <- c(2,3,5,7,11)
primes[primes == 4]

Another way to see if a value is contained inside a vector is using the %in% operator.

primes <- c(2,3,5,7,11)
4 %in% primes
primes <- c(2,3,5,7,11)
4 %in% primes

This works to compare each elements of a vector. Here it’s returning TRUE and FALSE for each element in odds, on whether or not it is also an element of primes.

primes <- c(2,3,5,7,11)
odds <- seq(1, 10, by=2)
odds %in% primes
primes <- c(2,3,5,7,11)
odds <- seq(1, 10, by=2)
odds %in% primes

TRUE and FALSE as binary indicators

What’s cool about TRUE and FALSE, is that TRUE resolves as 1 and FALSE resolves as 0 when doing arithmetic. How many odd numbers are prime (in our given vectors) 
primes <- c(2,3,5,7,11)
odds <- seq(1, 10, by=2)
sum(odds %in% primes)
primes <- c(2,3,5,7,11)
odds <- seq(1, 10, by=2)
sum(odds %in% primes)

AND and OR

Often we will want to find out if two events are true at the same time, or if at least one of them is true. Using parenthesis to help keep our statements organized, we can ask multiple logical statements at the same time using either AND & or OR |

Is 9 an odd prime?

 primes <- c(2,3,5,7,11)
odds <- seq(1, 10, by=2)
(9 %in% odds) & (9 %in% primes)
 primes <- c(2,3,5,7,11)
odds <- seq(1, 10, by=2)
(9 %in% odds) & (9 %in% primes)

You try it

Remember | means OR. Checkout how we answered the last question.

primes <- c(2,3,5,7,11)
odds <- seq(1, 10, by=2)
(9 %in% odds) | (9 %in% primes)

Plotting

For starters we are going to stick with the simple plotting method in R, namely the plot function. Short and simple, the arguments are plot(x, y)

x <- -10:10
y <- x^2
plot(x, y)
x <- -10:10
y <- x^2
plot(x, y)

The default is to just plot the points, but sometimes we may want to connect those points with a line (l is a lower case L). Type ?plot in the code box for more plotting types.

x <- -10:10
y <- x^2
plot(x, y, type='l')
x <- -10:10
y <- x^2
plot(x, y, type='l')
